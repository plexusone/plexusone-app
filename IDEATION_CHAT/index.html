    Let’s break it down in a practical way.

    🧠 1. How orchestrators like GSD actually work

    Projects like GSD are not “multi-agent magic”—they’re usually:

    Core loop

    Maintain a task graph / plan

    Spawn agents (CLI tools like Codex, Claude, etc.)

    Route:

    prompts

    outputs

    tool calls

    Persist state (files, logs, memory)

    Under the hood

    They rely on:

    subprocesses (spawn CLI tools)

    pseudo-terminals (PTYs)

    queues/event buses

    structured logs

    👉 In other words: GSD ≈ process manager + router + memory layer

    You’re thinking about this the right way—and the concern about token usage is very real. Most “multi-agent” systems (including things like get-shit-done) burn tokens not because they’re powerful, but because they’re poorly constrained.

    Let’s ground this in what’s actually happening.

    🔥 1. Why GSD-style systems burn so many tokens

    It usually comes down to 3 things:

    1. Over-communication between agents

    Agents “talk” too much

    Long context passed repeatedly

    Full transcripts forwarded instead of summaries

    👉 Token explosion pattern:

    Agent A → sends 2k tokens Agent B → replies with 3k tokens Agent C → gets both → now 5k tokens 2. No strict task boundaries

    Agents are told:

    “figure this out”

    Instead of:

    “do exactly this small step”

    So they:

    re-plan

    re-think

    re-explain context

    👉 That’s wasted tokens.

    1. No memory compression

    Full history passed every time

    No summarization layer

    No state extraction

    🧠 2. Why your tmux-first approach is actually smart

    Starting from manual tmux sessions gives you something most orchestrators lack:

    observability of real agent behavior

    You can literally see:

    what prompts work

    where tokens are wasted

    when agents go off rails

    👉 This is huge.

    Most frameworks skip this and go straight to abstraction.

    🧱 3. The principle you’re circling

    You’re basically saying:

    “Let’s understand the physics before building the operating system.”

    That’s exactly right.

    🧪 4. What you should study in your current setup

    tokens per response
